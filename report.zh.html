执行摘要

W3C于2021年6月25日组织了一场关于智慧城市的线上研讨会。此次研讨会的主要目标是改进并拟定关于潜在的智慧城市兴趣组的章程草案，以便我们可以启动兴趣组，并进一步讨论以下有关内容：一是基于Web的智慧城市服务的互通性；二是为支持智慧城市服务，W3C规范需满足的用例和要求。

在本次研讨会中，我们完成了以下工作：

  • 确定智慧城市标准化利益相关方，以推动符合智慧城市实际需求的Web标准的开发
  • 阐明智慧城市技术的合理应用
  • 了解如何改进潜在的智慧城市兴趣组的章程草案，以便在该兴趣组内部进一步开展讨论

我们还明确了采用包容性设计是非常重要的，包括可访问性、隐私性、安全性和国际化。智慧城市应用与居住在城市里的人们息息相关。我们希望智慧城市技术能对其生活产生积极影响，同时希望规避意料之外的负面后果。

研讨会后，W3C后续将通过敲定以下内容，完成最终版潜在的智慧城市兴趣组的章程草案：

  • 我们将提供的技术内容
  • 谁是参与潜在智慧城市兴趣组的主要利益相关者，谁将从小组讨论中受益
  • 哪些城市最先进、最适合参与试点

接着，我们将启动该兴趣组，并开始进一步讨论以下事项：

  • 针对智慧城市现有技术和标准的调研（技术前景）
  • 关于将哪些技术（如万维物联网、汽车制造、地理空间、VR/AR、语音和语义网等）应用于何种智慧城市应用的最佳实践案例，例如提高可访问性、访客引导和能源管理等
  • 智慧城市的用例和要求

为此，我们将与从事智慧城市工作的相关标准开发与发展组织密切联系，如OGC、ISO、IEC、ISO/IEC JTC1、ITU-T、BSI、ETSI、ECLASS和IETF等。此外，我们还将与相关的现有W3C小组合作，如万维物联网小组、分散标识符小组、媒体与娱乐小组、语音与Solid平台小组等。此外，直接向国家和社区提供服务也是重要且有益的，包括发展中国家和农村地区。世界经济论坛、东盟和智慧城市联盟可能是拓展这一领域的好伙伴。

有很多话题仍待讨论，预期的智慧城市兴趣组的范围可能会变得过于广泛，鉴于研讨会期间的讨论情况，或许第一年应该专注于数据治理和隐私管理。

研讨会会议

由于参会人员来自不同地区，研讨会由三场单独的会议组成。这种会议形式非常高效，能够让所有演讲者和其他与会者都可以参与研讨会讨论，而不受时区的限制。所有会议内容均已录制。

三场会议均由以下子会议组成：

  • 场景设置：研讨会的目标和期望
  • 用例
  • 现有标准
  • 基于Web的方法的建议
  • 智慧城市的跨领域问题
  • 总结

在上述每场子会议中，专门讨论了对智慧城市兴趣组章程草案最重要的要求。在每次会议结束时，均总结了我们在研讨会期间讨论的内容。此外，在最后的第三次会议结束时，明确了需要一个专门的讨论小组来进行进一步讨论，以及需要在智慧城市兴趣组章程草案中添加哪些内容。

以下为各次会议的摘要。另请参阅研讨会的详细会议记录。

会议（一）：6月25日06:00-07:35（UTC时间）

1. 场景设置

研讨会主席和工作人员联系人阐述了研讨会的目标和期望，即本次研讨会旨在敲定拟议的兴趣组章程，并要求所有参会者专注于此目标。

2. 智慧城市现状总结 — 用例

进行了以下两场演讲：

  • Heng QIAN: 智慧城市信息与通信技术的独特之处
  • Peter Lee: 智慧萨福克郡

接着，我们进行了讨论，从用例的角度出发，明确了智慧城市兴趣组章程草案中需要补充的内容:

  • 将已明确致力于智慧城市建设的标准开发与发展组织添加到章程草案的联络部分，包括ISO、IEC、ISO/IEC JTC1、BSI、ETSI。
  • 将W3C在当前全球环境中的角色在范围部分予以澄清。
  • 需收集用例，据此分析需标准化的内容，而数据治理是实现这一目标的关键之一。
  • 应同时考虑国际标准和国内标准。
  • 隐私标准和隐私管理非常重要，需考虑信任谁，以及如何表示及衡量信任度。
  • 其中一个可能的用例是公共交通服务的“最后一英里”，例如，从家到火车站的距离，应考虑包括哪些区域。
  • 数据互操作性也很重要，建议使用一些高级数据分类和模式来保证元数据的一致性。
  • 如何在智慧城市环境中的各个利益相关者之间达成一致很重要。还需要考虑谁/如何发挥领导作用。
  • 应该考虑数据治理的各种详细观点，且需要制定相应框架。

3. 基于Web的智慧城市的需求和可能的解决方案

主题演讲：

  • Sebastian Kaebisch: 智慧城市中的标准化服务编排

然后，我们进行了讨论，从基于Web的方法的角度出发，明确了 智慧城市兴趣组章程草案中需要补充的内容：

  • 应将当前的情况总结成为智慧城市的前景资料，总体概述已经可用的技术类型。
  • 预期的“概览文件”应充分涵盖智慧城市的用例、技术和利益相关者。还应阐述所有标准的基本通用问题，如集成。
  • 应阐明 W3C在智慧城市概览中的角色，例如智慧城市服务的用户界面和消费层，以改善用户体验。

4. 智慧城市的跨领域问题

主题演讲：

  • Sisay Chala, Otilia Werner-Kytölä: 智慧城市背景下的隐私意识信息库

然后，我们进行了讨论，从跨领域问题的角度出发，明确了智慧城市兴趣组章程草案中需要补充的内容:

  • 提出了需要具备隐私意识的信息管理框架。
  • 在这里，隐私和可用性之间的平衡很重要，例如地理位置数据。
  • 还应该谨记现有的法律框架，例如《通用数据保护条例》等法规。

5. 总结

我们总结出以下在智慧城市兴趣组章程草案中待补充的内容:

  • 在“4.2外部组织”一节中，应添加与从事智慧城市工作的国际/国内标准开发与发展组织的联络。此外，应该在“4.1 W3C组”一节中添加更多的W3C内部协作内容。
  • 不仅要考虑数据传输，还要考虑数据与实体的结合。
  • 应基于智慧城市的具体用例讨论需求。
  • 应阐明现有标准、技术和系统的概况和前景。
  • 应考虑可用性和隐私保护之间的平衡，特别是在地理位置和时间等敏感信息上。另外，还应将其视为一个整体机制，其服务应作为数字基础设施。
  • 还应考虑智慧城市的可访问性和数据治理。
  • 应明确W3C在智慧城市环境中的角色，例如，为智慧城市服务提供易于理解和使用的交互界面。
  • 还应确定智慧城市的用例，查看哪些Web技术是适用的。然后评估用例和现有Web技术(包括浏览器技术和其他Web技术)之间的差距，进行优先排序。

会议（二）：6月25日12:00-14:10（UTC时间）

1. 场景设置

研讨会主席和工作人员联系人阐述了研讨会的目标和期望，即本次研讨会旨在敲定拟议的兴趣组章程，并要求所有参会者专注于此目标。

2. 智慧城市现状总结 — 用例

主题演讲：

  • Josh Lieberman: 城市数字孪生的社会化
  • Daihei Shiohama: 日本智慧城市的WoT用例

接着，我们进行了讨论，从用例的角度出发，明确了智慧城市兴趣组章程草案中需要补充的内容：

  • 由于空间和时间信息的作用，智能城市中的数字孪生比普通的物联网环境更复杂，例如，应考虑社会和物质之间的平衡。
  • 数字化转型正在冲击包括出版业、音乐业在内的各个行业，因此为了利用Web技术振兴实物产业，有必要对实物服务和数字化服务的优缺点进行分析。为此，我们应该考虑零售服务的用例，包括实体服务和数字化服务。
  • 地理空间信息至关重要。然而目标不仅是物理空间，也包括数字空间。因此，我们需要一个既可以应用于物理空间，也可以应用于数字空间的空间信息社会模型。
  • 应明确Web技术的角色，包括Web浏览器技术和WoT技术。

3. 智慧城市现状总结 — 现有标准

主题演讲：

  • Jerome Blum: 智慧城市的标准化分类、术语及语义

接着，我们进行了讨论，从智慧城市相关现有标准的角度出发，明确了智慧城市兴趣组章程草案中需要补充的内容：

  • 不同行业的数据定义互操作性很重要，我们应与包括ECLASS、IEC CDD和ISO TC184/SC4在内的数据定义标准开发与发展组织合作。
  • 还应考虑对数据管理的信任和可能的机制来保证信任，比如Solid。

4. 基于Web的智慧城市相关需求及可能的解决方案

主题演讲：

  • Michael McCool: 智慧城市中的万维物联网
  • Andrea Cimmino: 使用Web技术从智慧城市转变为智慧社区
  • Jacqueline Lu: 日常场所的透明界面
  • Dave McComb: 从企业本体到智慧城市的经验

然后，我们进行了讨论，从基于Web的方法的角度出发，明确了智慧城市兴趣组章程草案中需要补充的内容:

  • 万维物联网中已有智慧城市的用例，例如访客引导、能源管理和健康/零售服务，应避免为物联网目的而重新创造。
  • 在智慧城市的背景下，个人数据管理存在很多问题。
  • 服务的发现是关键，而地理位置信息对于物理服务来说是很重要的。然而经纬度信息并不是空间的唯一表示，空间的概念也存在差异，例如邻域。
  • 应考虑以人为本的观点以及社会互动，还需要解决通讯问题。
  • 智慧城市的数据治理也应在基于Web的方法中加以阐明。
  • 为此，我们应考虑各种用例，涵盖智慧城市中的多个行业领域，例如停车和消防服务，并考虑其中数据传输的权责如何划分。
  • 当前已有数据通信和数据表示的标准，因此我们应与研究这些标准和机制的标准开发与发展组织和项目联系，例如在纽约、波士顿和多伦多附近的那些。
  • 对于系统实现，我们应考虑复用现有的稳定机制，避免重复开发。
  • 开放地理空间联盟已被列入“4.2外部组织”中，但应考虑开展各个角度的深入合作，包括GeoPose在内。
  • W3C已完成许多可能的技术组件，例如RDF、OWL、SHACL、ODRL 和 WoT。另一方面，实际的词汇定义是在W3C之外完成的，例如由ETSI实现的SAREF本体。因此我们应与这些相关的标准开发与发展组织合作，了解如何为基于Web的智慧城市集成必要的组件。

5. 智慧城市的跨领域问题

主题演讲:

  • Deborah Dahl: 智慧城市的智能用户界面

然后，我们进行了讨论，从跨领域问题的角度出发，明确了智慧城市兴趣组章程草案中需要补充的内容：

  • 应为智慧城市的访客提供智能对话界面，以确保多个系统之间的互通性和可达性。
  • AR/VR界面可能和语音界面一样有用，但需要注意的是，不同的人根据自身情况需要不同类型和形式的界面。基于图标的方法可能是个很好的起点。
  • 不同城市有着不同的优先级，也应考虑发展中国家的智慧城市问题。
  • 应该将语音交互社区组添加到“4.1 W3C组”中，将开放语音网络添加到“4.2外部组织”中，以便联络。

6. 总结

我们已确认，本次会议提出的兴趣组章程草案中所缺少的全部内容，都已在上述讨论中明确。

会议（三）: 6月26日01:00-02:25（UTC时间）

1. 场景设置

研讨会主席和工作人员联系人阐述了研讨会的目标和期望，即本次研讨会旨在敲定拟议的兴趣组章程，并要求所有参会者专注于此目标。

2. 智慧城市现状总结之现有标准

主题演讲：

  • Clarissa Loureiro: 发展中国家智慧城市成熟度模型

接着，我们进行了讨论，从智慧城市相关现有标准的角度出发，明确了智慧城市兴趣组章程草案中需要补充的内容:

  • 巴西有一个基于ITU-T标准的关于可持续智慧城市模型的政府项目。由于互联网连接质量等原因，ITU-T标准被扩展以适应拉丁美洲国家的情况。
  • 应考虑与ITU-T、ISO等相关标准开发与发展组织进行联络，并考虑如何将现有标准应用到包括发展中国家在内的各个国家。此外，应考虑城市、国家和文化之间的差异。
  • 我们不仅应该关注整个城市，还应该关注居住在城市中的人们，以看到高效、优质的公共服务。为此，应开展最佳实践案例及现有标准调查，以适应包括发展中国家和农村地区在内的各个国家和地区。
  • 我们应与各个区域智慧城市组织合作，如亚洲的东南亚国家联盟等，也应与ITU-T、ISO和ISO/IEC JTC1等标准开发与发展组织合作。
  • 我们还应该注意，在智慧城市和潜在的智慧城市中从事公共服务的人们可能并不具备足够的技术水平。因此，根据每个国家的情况进行一些教育或培训，了解优先事项很重要。
  • 特别是在新冠肺炎疫情下，网络连接是智慧城市非常重要的基础设施。所以我们也应考虑当前形势对每个国家/地区的影响。例如，电视广播可能是教育和培训的一种解决方案，但并不是每个人都有条件参与。
  • 应考虑包括硬件、中间件和网络（即互联网）在内的多层机制。即使该机制的底层部分超出了W3C的标准化范围，我们也应该收集整理关于“哪些层缺少什么”、“我们可以做些什么来改善智慧城市景观”的信息。
  • 为此，W3C内部开展深入的合作是有必要的，例如在网络上的流媒体和电视服务方面与媒体和娱乐兴趣组展开合作。
  • 我们应在与政府、社区和标准化组织讨论的基础上收集信息和需求，以了解智慧城市的实际需求。此外，我们应阐述智慧城市标准化工作的重要意义，以说服重要利益相关者参与其中。
  • 我们需要某种机制来联系重要利益相关者，W3C分会机制能够实现此目的。

3. 智慧城市的跨领域问题

主题演讲：

  • Baoping CHENG: 多媒体通信技术重塑智能家居生活
  • Kaz Ashimura: 智慧城市的数据治理

然后，我们进行了讨论，从跨领域问题的角度出发，明确了智慧城市兴趣组章程草案中需要补充的内容：

  • 将人工智能技术应用于物联网将有助于顺利部署。可以使用智能语音和面部识别来进行身份验证和访问赋权。
  • 关于访问权限的管理，应与相关的W3C组合作，包括分散标识符工作组、Web认证工作组和Solid社区组。
  • 正如维基百科中所描述的智慧城市，智慧城市有各种优点和缺点，并对其中的隐私管理提出了一些批评，例如高水平的大数据收集和分析。
  • 对于真正的智慧城市服务，我们需要思考如何管理和治理我们自己的数据。在智慧城市环境中有太多的利益相关者，但我们应注意以用户为中心，并考虑如何管理包括我们自己在内的所有利益相关者之间的数据传输。
  • 为此，我们需要根据具体的用例和需求，结合自身情况来确定在复杂的环境中应该做什么。

4. 总结

我们快速回顾了关于应添加到智慧城市兴趣组章程草案中的重要要求的讨论：

  • 我们收到建议，应适应各种现有技术来提供智慧城市服务，例如物联网、人工智能、AR/VR、语音、地理空间和语义网等。
  • 明确了安全性、隐私性、可访问性和国际化（适应不同地区的文化和政策）对智慧城市而言很重要。我们应始终以用户为中心。目前的智慧城市兴趣组章程草案更偏向于技术导向,因此我们应根据本次研讨会的讨论，考虑使其更加人性化，例如支持社交、互动,不仅提供硬件和设备的观点,也要涵盖媒介和内容。
  • 确定将扩展与其他标准开发与发展组织的联系，例如OGC、ISO、IEC、ISO/IEC JTC1、ITU-T、ETSI、 ECLASS等等。也看到了与现有W3C组织(如万维物联网小组、分散标识符小组、媒体与娱乐小组、语音与Solid平台小组)进一步深入合作的重要性。此外，直接向国家和社区提供服务也是重要且有益的，包括发展中国家和农村地区。世界经济论坛、东盟和智慧城市联盟可能是拓展这一领域的好伙伴。我们面临的挑战是如何对所有候选组织进行优先排序。我们可以利用W3C分会机制实现这一目的。

5. 后续工作

  • 本次研讨会的主要目标是收集意见并改进智慧城市兴趣组章程草案。
  • 作为研讨会讨论的结果，我们达成了一个共识，即我们应在与来自不同领域的利益相关者进行更广泛的讨论的基础上，开始实际讨论基于 Web 的智慧城市的预期用例和要求，并在W3C内部建立一个专门的讨论组。
  • 关于智慧城市兴趣组章程草案的范围，我们确定了主题和重点内容，包括数据治理、发展中国家的智慧城市以及对现有智慧城市的调查。
  • 我们还确定了来自其他重要的社会服务机构、政府机构、智慧城市项目和社区的潜在联络人，邀请他们成为研讨会的演讲人，包括来自Josh Lieberman (OGC), Heng Qian (ISO/IEC JTC1), Peter Lee (BSI), Jerome Blum (ECLASS), Sisay Chala (GOEASY), Jacqueline Lu (DTPR), Dave McComb (semantic arts), Clarissa Loureiro (Clarissa Loureiro)。
  • 我们将根据研讨会的反馈最终确定智慧城市兴趣组章程草案，并启动该小组，除了对智慧城市现有的技术和标准进行调查之外，还会针对智慧城市的用例和需求展开进一步讨论。

W3C 是一个开放包容的组织，专注于高效的讨论以及富有成效的行动。 我们的《 职业道德与行为准则》确保我们积极听取来自各方的意见与声音。如有任何疑问，欢迎联系 Kaz Ashimura <ashimura@w3.org>。
