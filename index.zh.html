背景

“智慧城市”通常指涉及建筑与居住环境的智能管理的一系列技术和流程。随着各方兴趣渐浓，我们看到了更智能与更轻便地集成 IoT 设备与 Web 服务的多个厂商的不同技术的未来发展趋势。

几个 智能城市的用例 已经被万维物联网 Web of Thing (WoT) 兴趣组作为 WoT 标准的一部分在 慕尼黑第二届 WoT 研讨会提案 中进行了讨论。然而，智慧城市还包含了各种技术，WoT 只是其中之一。我们因此希望能创建一个专门的 W3C 兴趣组来：

  • 界定和记录用例与需求从而让 W3C 标准支持智慧城市服务，
  • 从各方参与者中收集使用 Web 技术建设智慧城市的反馈建议，
  • 收集各方关于智慧城市所需的 Web 技术属性的专业意见，并且
  • 提供平台进行智慧城市相关的技术和商业讨论。

我们以及起草了 策划中的智慧城市兴趣组的章程初稿。我们希望从世界各地的厂商那里收集更多的用例与具体的实现经验，因为其使用可能依赖于城市的地点、文化等等，并包括来自许多不同供应商的各种子系统讨论。因此，我们想举办一次专门的 W3C 研讨会，以探讨智慧城市可能实现的标准化。

城市通常包含了不同的人群，包括残障人士，我们应该在智慧城市建设中融入无障碍思考、隐私、安全和多语言设计等考虑因素。

目标

Web 已成为各种行业和服务的平台，包括支付和商业、出版、媒体发布、视频会议和车联网，而且它成为值得期望的实现 IoT 互操作性的平台，如 2020年四月发布的 Web of Things 媒体报告所说的。此外，还有期望 Web of Things 标准能被应用于联通大量的智慧城市服务子系统。我们起草了一份 智慧城市兴趣组的章程初稿 来 (1) 引入 W3C 作为一个讨论智慧城市服务互通性的地方并且 (2) 记录用例与需求来让 W3C 标准支持智慧城市服务。

现在我们希望能完善和定稿 兴趣组章程草案。在本次 W3C 研讨会，我们希望看到现有的标准能达成的技术突破，以及实现智慧城市还缺失的支持性技术。

本次智慧城市研讨会的目标是：

  • 确定智慧城市标准化的利益相关者，以同步探索智慧城市的实际需求与开发标准
  • 弄清楚我们能达成共识构建的合理的智慧城市应用技术
  • 并且优化将来要创建的智慧城市兴趣组建议章程草稿

智慧城市的应用与生活在城市的人密切相关，并且有可能会对他们的生活产生负面影响。 因此，从一开始就加入包容性设计、无障碍思考、隐私和安全等考虑因素是非常重要的。

话题

研讨会将可能涵盖以下话题：

  • 总结智慧城市现阶段的发展情况：
    • 我们以及达成的
    • 哪些标准可以专门用于智慧城市场景
    • 不同厂商的产品实现互联互通的共通难题
    • 现有的技术机制的缺失属性
  • 来自 Web-基础的智慧城市的参与者，包括用户、开发者和政府的需求：
    • 更加智能和轻便的互通性
    • 优化/拓展现有的服务
    • 应用先进技术如：机器人、语音识别、AI 等等，到智慧城市用例中
    • 处理人类、设备、地点、环境等相关信息的必要元数据 metadata
  • 多个子系统互联互通的跨领域问题：
    • 无障碍，国际化，安全与隐私
    • 数据可信任与数据治理
    • 资源发现
    • 分类和本体论的语义和语法定义
    • 不同数据格式/模型的数据的语义互通性
    • 实时互动的高速网络环境

可能的用例的示例

智慧城市的可能的用例应该包含多种设备和由不同厂商开发的子系统的互通性，也应该考虑不同参与方的立场。

下面列举了一下从用户角度的用例：

  • 支持所有的群体
    • 允许残障人士使用的包容性设计
    • 多语言和多方式支持
    • 整个使用周期的安全与隐私保护
    • 备灾和应急响应
    • 整个使用周期的监控与保护
  • 交通与车联网
    • 辅助无障碍操作的自动驾驶
    • 无障碍共享汽车服务
    • 交通管理
  • 教育
    • E-learning 和教育机构线上化
    • 无障碍课堂
    • 成人持续教育
  • 健康
    • 传染病预防
    • 远程手术
    • 日常健康检查
  • 信息建设
    • 通用的网络帮助关闭数字鸿沟
    • 5G 移动网络

什么是 W3C？

W3C 是一个世界范围内开放的标准组织，汇集众多公司与社区成员共同围绕现有和新兴技术展开富有成效的讨论，其为 Web 标准设立了免版税的专利框架。W3C 主要关注客户端（浏览器）技术，并且拥有成熟的词汇表（或“本体”）开发历史。W3C 根据成员和社区输出的优先级开展相应标准化工作。

W3C 是一个开放包容的组织，专注于高效的讨论以及富有成效的行动。 我们的 职业道德与行为准则 确保我们积极听取来自各方的意见与声音。 有任何问题，请联系 Kaz Ashimura <ashimura@w3.org>.
